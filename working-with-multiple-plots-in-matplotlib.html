Working with multiple plots in matplotlib

Sun 29 July 2018

This is part of a series of posts about plotting using matplotlib in python.

subplots()

Often it is helpful to create several plots arranged in a grid, and matplotlib provides easy functionality for doing so.

The function which can be used for this is also an easy way to create a figure, and this is what I most commonly use for anything more complex than a very basic plot. Becuase this is so common, I'll start by describing the use of subplots() to create a single figure, and then go on to describe creating multiple subplots in a single figure.

The basic command for either case is subplots(). This returns two objects -- a Figure and an Axes object (or an array of Axes objects).1 The Figure is required for certain configurations, but in many cases you'll only need to make use of the Axes object.

For example, rather than using the plt.plot() command directly, you can first create the figure, and then plot on the corresponding Axes.

In [4]:
%matplotlib inline
import numpy as np
import matplotlib.pyplot as plt

# make some data
x = np.arange(-10, 11)
y = x ** 2

# create the figure
fig, ax = plt.subplots()

# plot the data
ax.plot(x, y);

Any arguments you would use in calling figure() can be passed to subplots(), the most useful is figsize, which allows you to specify the figure size as a (width, height) tuple.

Many of the functions which you might be used to calling directly, such as title() or xlim(), you can access using in a slightly modified form from the Axes object directly. For example, rather than calling plt.title('Title'), you would use ax.set_title('Title'). The same pattern holds for xlabel, xlim and so on, which are called using ax.set_xlabel() and ax.set_xlim() respectively.

Multiple plots

To create multiple plots, in a grid, you can specify the number of rows and columns. These are the first two positional arguments, so just providing a single number will create a columns of figures.

In [5]:
%matplotlib inline
import matplotlib.pyplot as plt

fig, ax = plt.subplots(3, figsize=(2, 6))
plt.show();

When you create multiple figures in this way, the second object that is returned will be an array of Axes object. When there are multiple rows and columns, it will be a two-dimensional array.

In [2]:
%matplotlib inline
import matplotlib.pyplot as plt

fig, axes = plt.subplots(nrows=2, ncols=2)
(ax1, ax2), (ax3, ax4) = axes
ax1.text(0.4, 0.4, 'ax1', fontdict={'size': 14})
ax2.text(0.4, 0.4, 'ax2', fontdict={'size': 14})
ax3.text(0.4, 0.4, 'ax3', fontdict={'size': 14})
ax4.text(0.4, 0.4, 'ax4', fontdict={'size': 14});

It's possible to make a bunch of plots that are totally independent of each other, but if they are going to be on the same scale, you may want to share the axes (that is the x-axis and the y-axis, not to be confused with the Axes object). In this way, the x and y limits of all plots will be adjusted to fit all of the data as necessary.

In [10]:
%matplotlib inline
import matplotlib.pyplot as plt
import numpy as np

fig, axes = plt.subplots(nrows=2, ncols=2, sharex=True, sharey=True)

for i in range(2):
    for j in range(2):
        x = np.linspace(-3, 3, 20)
        y = np.power(x, i*2+j)
        axes[i][j].plot(x, y)

You've also probably noticed that subplots() puts space between the figures by default. If you want to get rid of that, you'll have to make an adjustment to the Figure object, using the subplots_adjust() function. Somewhat confusingly, the arguments to that function are called hspace (for height) and wspace (for width).

In [2]:
%matplotlib inline
import matplotlib.pyplot as plt
import numpy as np

fig, axes = plt.subplots(nrows=2, ncols=2, sharex=True, sharey=True)

for i in range(2):
    for j in range(2):
        x = np.linspace(-3, 3, 20)
        y = np.power(x, i*2+j)
        axes[i][j].plot(x, y)

fig.subplots_adjust(hspace=0, wspace=0.5)

If you want to get rid of one of the subplots all together, you can also do so using the ax.axis('off') function.

Advanced

Finally, it's possible to get quite fancy and build a plot with differently sized subplots, or even subplots with take up different portions of a grid. For this, it's often easier to start with an emtpy Figure, and add Axes to it one at a time. The example below will give you some sense of what is possible.

In [8]:
%matplotlib inline
import numpy as np
import matplotlib.pyplot as plt
from matplotlib.gridspec import GridSpec

# create an empty figure
fig = plt.figure()

# specify 2 rows and 3 columns, with a 3:1:1 width ratio
gs = GridSpec(nrows=2, ncols=3, width_ratios=[3, 1, 1]) 

# add Axes to the Figure in specified locations
ax1 = fig.add_subplot(gs[0, 0]) # First row, first column
ax2 = fig.add_subplot(gs[0, 2]) # First row, third column
ax3 = fig.add_subplot(gs[1, :]) # Second row, span all columns

# plot some data
x = np.linspace(-3, 3, 20)
ax1.plot(x, x**2)
ax2.plot(x, -x*2)
ax3.plot(x, np.sin(x));

Happy plotting!

  1. This terminology is unfortunately quite confusing, but within matplotlib, the name Axes refers to the plotting space, i.e. a container for everything you will put on the plot. The Figure contains the Axes (or multiple Axes), along with the legend, etc. By contrast an Axis refers to the lines with ticks along the sides and bottoms. ↩
