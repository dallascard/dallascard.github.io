Building this Blog

Sun 20 November 2016

Getting set up on Github

On Github, create a new, empty, public repo: \<username>.github.io

Create a second repo to keep source files (this one can be public or private, and you can add a Python .gitignore)

Clone the second repo and cd into it.

Creating a static blog with pelican

Install pelican and markdown:

$ pip install pelican
$ pip install markdown

Run pelican-quickstart. You can enter a name, author and time zone, all of which can be changed later. Otherwise, you can accept the defaults everywhere except where for specifying a URL prefix which should be "n".

$ git clone https://github.com/dallascard/github_blog.git
$ cd github_blog
$ pelican-quickstart
Welcome to pelican-quickstart v3.6.3.

This script will help you create a new Pelican-based website.

Please answer the following questions so this script can generate the files
needed by Pelican.


> Where do you want to create your new web site? [.] 
> What will be the title of this web site? Github Blog
> Who will be the author of this web site? Dallas Card
> What will be the default language of this web site? [en] 
> Do you want to specify a URL prefix? e.g., http://example.com   (Y/n) n
> Do you want to enable article pagination? (Y/n) 
> How many articles per page do you want? [10] 
> What is your time zone? [Europe/Paris] US/Pacific
> Do you want to generate a Fabfile/Makefile to automate generation and publishing? (Y/n) 
> Do you want an auto-reload & simpleHTTP script to assist with theme and site development? (Y/n) 
> Do you want to upload your website using FTP? (y/N) 
> Do you want to upload your website using SSH? (y/N) 
> Do you want to upload your website using Dropbox? (y/N) 
> Do you want to upload your website using S3? (y/N) 
> Do you want to upload your website using Rackspace Cloud Files? (y/N) 
> Do you want to upload your website using GitHub Pages? (y/N) 
Done. Your new project is available at /Users/dcard/Projects/blogs/github_blog

This will create a series of subdirectories. The most important files and folders to know about are the content subfolder, where you will place markdown posts to be rendered, and the pelicanconf.py file, where you can control various aspects of the blog (including name, author, etc.)

Creating content

Make a first post in the content subdirectory:

$ cd content

Create a new file called first_post.md and add the following lines to it. (Make sure there are no blank lines at the top).

Title: first_post
Date: 2014-12-13 18:32
Category: Test

Hello world from Pelican!

Return to the main directory

$ cd ..

Building and serving the blog locally

To build the blog

$ make html

To serve the site (to http://localhost:8000)

$ make serve

You should now be able to view the blog in a browser at http://localhost:8000

To shutdown the server, use Ctrl-c

Optionally, you can run make regenerate to start a process that will automatically rebuild the site upon changes, or a single process that will automatically rebuild and serve the site (make devserver). To stop make devserver, use:

$ ./develop_server.sh stop

Changing themes

To change themes, fork and clone the repo of pelican themes (https://github.com/getpelican/pelican-themes) or a specific theme. We'll go with pelican-octopress-theme from https://github.com/duilio/pelican-octopress-theme. (Remember to fork the repo to be able to make and save changes).

Add the theme to the list of pelican themes:

$ pelican-themes --install pelican-octopress-theme

In the main directory, edit the pelicanconf.py by adding the following line:

THEME = "pelican-octopress-theme"

The blog should now reflect the new theme when you rebuild and reload the page. (Note that sometimes my Chrome plugins caused the theme to fail to load properly at localhost:8000; try using another browser for displaying locally).

Using plugins

A large set of plugins enhance pelican's functionality, for example, displaying equations.

Clone the pelican plugins repo (https://github.com/getpelican/pelican-plugins)

Edit the pelicanconf.py file to add the following lines:

PLUGIN_PATHS = ['../pelican-plugins/']
PLUGINS = ['render_math']

\(\LaTeX\) math should now render properly: \(x_i^2 = \sin(y)\)
