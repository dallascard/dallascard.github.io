Swapping the y-axis in matplotlib

Sun 04 March 2018

Partly as notes for myself, I've decided to start posting snippets of code for various plotting functionality in python's matplotlib.

Matplotlib is quite powerful, and can be used to make a huge variety of plots, but it can also be frustrating, espeically when you're trying to figure out how to do some specific thing.

The answers can almost always be found somewhere online, but I thought I would post things here that I personally tend to forget.

Basic plotting in an iPython notebook

This is hopefully well known to most people, but to use matplotlib in an iPython notebook, start with the inline "magic" command, and then import pyplot from matplotlib.:

%matplotlib inline
import matplotlib.pyplot as plt

The best practice is to start by making a figure and a set of axes with the plt.subplots() function.

fig, ax = plt.subplots()

This function returns two objects: a handle to the figure, and a collection of one or more subfigures, which we'll call axes. (This can also be extended to make a figure with multiple subplots). This allows us to easily specify details, such as setting the limits or labels. Here is a first basic example:

In [16]:
%matplotlib inline
import matplotlib.pyplot as plt
import random
n = 100; x = range(n); y = [i**2 for i in x]

fig, ax = plt.subplots()
ax.plot(x, y);

(The semi-colon prevents the iPython notebook from printing the value that is returned by the last line.)

Moving the y-axis to the right side of the plot

By default, the y-axis is on the left side of the plot. In some circumstances, we might prefer to have it on the right, and naturally matplotlib can do this for us. The key is to create a second axis, which is a twin of the first. This can be done using:

fig, ax_left = plt.subplots()
ax_right = ax_left.twinx()

To plot data such that the y-axis will appear on the right, we now need to plot it on the new axis, not the old one, so we'll use, for example,

ax_right.plot()

This will cause the y-axis to appear on the right, but the figure will now have two y-axes, one on the left, and one on the right. To avoid confusion, the simplest thing is to simply remove the ticks for the y-axis on the left, by setting them to an empty list:

ax_left.set_yticks([])

Of course, we could keep both axes and simultaneously plot some data on the left axis and some on the right, using different scales. Although you do see this on occasion, it can be confusing, and is generally bad practice.

Here's a complete example:

In [6]:
%matplotlib inline
import matplotlib.pyplot as plt
import random
n = 100; x = range(n); y = [i**2 for i in x]

fig, ax_left = plt.subplots()
ax_right = ax_left.twinx()
ax_right.plot(x, y)
ax_left.set_yticks([]);
